## 带你了解函数式编程 
### 简介

下面的内容会包含比较多的术语，但都会给予明确的概念和解释。

函数式编程（英语：functional programming）是一种编程范式，即如何编写程序的方法论。它以`λ 演算（英语：lambda calculus，λ-calculus）`为理论基础，换句话来说函数式编程实现了 λ 演算支持。

### λ 演算

[λ 演算](https://zh.wikipedia.org/wiki/Λ演算#解释与应用)是一套从数学逻辑中发展，来`研究函数`如何进行抽象化定义、如何被应用以及递归的形式系统。定义比较抽象，简而言之就是`研究函数`形成的一套方法。

#### λ 演算的语法

下面以表格的方式说明一些语法概念，方便后续的理解和使用:

|  语法  |  名称  | 描述                                                                  |
| :----: | :----: | --------------------------------------------------------------------- |
|   x    |  变量  | 用字符或字符串来表示参数或者数学上的值或者表示逻辑上的值              |
| (λx.M) | 抽象化 | 表示函数定义（M 是一个 lambda 项），在表达式中的 x 都会绑定为变量 x。 |
| (M N)  |  作用  | 作用单个的函数 M 到单个的自变量（argument）N。 M 和 N 是 lambda 项。  |

> 如果在“作用”上有歧义，我们可以使用(M)(N)的语法。

#### λ 演算的文字描述

在下面的形式化表达之前，我们希望通过文字结合 λ 演算 的语法，来加深对其概念应用的理解。

因为在解读上，λ 演算的文字描述要比普通函数表达式的文字描述要更为抽象些，所以跳过对λ 演算的文字描述的理解，也就不能透彻理解基于 λ 演算 理论而来的编程概念。

有一个函数表达式 $f(x) = x + 2$，用 λ 演算 的语法表示是 $λx.x + 2$（x 可以任意命名）。我们以 λ 演算的 (λx.M) 语法为文字描述就是“**定义了一个函数 M （指代 x + 2），M 里的 x 都会绑定为变量 x**”。

而$f(3)$的值用 λ 演算的语法可以写作 $(λx.x + 2)\ 3$，也可以写作：$(λx.x + 2)(3)$，即 (M N) 的语法格式。转为文字描述就是“**把函数 M 作用于参数为 3 的值上**”。

考虑一个`抽象化的函数`：它把一个函数作为参数，这个函数将被作用于 3 上。于是我们可以表示成：$λf.f\ 3$。接下来我们把这个函数作用于上面的 $λx.x + 2$ 上，于是形成的函数就是 $(λf.f\ 3)(λx.x+2)$。

由此可以推导出下面三个表达式是等价的：

$(λf.f\ 3)(λx.x+2)$ 等价于 $(λx.x + 2)\ 3$ 等价于 $3 + 2$

这里有了一个新的概念：当一个单一参数的函数，它的返回值又是一个单一参数的函数，这样的函数称之为`柯里化函数`。例如：一个多参函数 $f(x, y) = x - y$ 柯里化后可以写作：$λx.(λy.x - y)$，去除括号即 $λx.λy.x - y$。

由此可以推导出下面三个表达式是也是等价的：

$(λx.λy.x - y)\ 7\ 2$ 等价于 $(λy.7 - y)\ 2$ 等价于 $7 - 2$

#### λ 演算的形式化

有其特定语法的表达式，即 lambda 表达式。每个 lambda 表达式都是一个匿名函数，并只有一个参数和一个返回值，同时无论参数还是返回值都是一个单参的函数。
